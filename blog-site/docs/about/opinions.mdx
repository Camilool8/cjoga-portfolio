## On Kubernetes

**It's not over-engineering for ~10 services on three nodes.** It's the cheapest way to get rolling deploys, health checks, secret management, and ingress routing that exists. The complexity people complain about is mostly cloud-provider integrations you can skip on K3s.

**etcd vs. SQLite** — pick etcd from day one even on a single-node cluster (`--cluster-init`). Migration later is a tax I paid and you don't have to.

**Service meshes are overkill** for anything under 50 services. The observability you actually need (traces, latency, errors) you can get from OpenTelemetry + a backend without Istio's operational cost.

## On infrastructure as code

**Terraform is the right default** for cloud infrastructure. Pulumi is more pleasant if your team is JavaScript-first, but the ecosystem and module quality is still Terraform's game.

**Stop checking `terraform.tfstate` into S3 without locking.** Just use the [HCP Terraform free tier](https://www.hashicorp.com/products/terraform) or a backend that supports state locking natively. Past-me has bricked things this way.

**Ansible is for configuration, not orchestration.** The moment you find yourself writing `wait_for` loops, you want a different tool.

## On observability

**You can't optimize what you don't measure** is a cliché because it's true. But the inverse — measuring everything — wastes your time too. Start with:

1. **Uptime** (synthetic checks)
2. **Latency** (p50, p95, p99 on the critical paths)
3. **Error rate**

Add more only when you've used the first three to fix something.

## On certifications

They're worth doing **if you actually learn the material**, not if you cram a brain dump.

**Highest ROI in 2026**:

1. **CKAD / CKA / CKS** — actually changes how you think about Kubernetes
2. **HashiCorp Terraform Associate** — forces fluency in HCL
3. **AWS Solutions Architect (Associate)** — the cloud lingua franca

**Skip** anything that's pure trivia. The good certifications are hands-on labs.

## On AI in DevOps work

Useful for:

- Drafting Terraform modules from natural-language specs
- Explaining unfamiliar error messages
- Generating boilerplate Kubernetes manifests
- Code review at the "did I forget a check" level

Not yet useful for:

- Real architectural decisions
- Anything where it can't see the production state
- Convincing security people to give it cluster credentials

## On working remotely from the Dominican Republic

It works. The internet here is fine. The talent pool is excellent. The biggest friction is time-zone alignment with European clients — solvable with discipline.

Most clients pay USD wages for DR cost-of-living, which is one of the better arbitrage opportunities in tech if you can build a portfolio that proves you're the equal of any onshore engineer.

The way to prove that is to **build things in public**. Hence the homelab. Hence this handbook.
